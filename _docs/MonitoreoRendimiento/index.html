Reporte de Monitoreo y Optimización de Rendimiento Frontend

Auditoría, justificación y plan de optimización para llevar al 100% las métricas de Lighthouse.

1. Diagnóstico Inicial y de Entorno

⚠️ Hallazgo Crítico de Medición

Las pruebas iniciales arrojaron métricas deficientes (Performance 32%-42%, LCP hasta 43s).

  • La Causa: Pruebas ejecutadas contra el dev-server de Vite (puerto 5173).
  • El Problema: Sirve archivos individuales, sin minificar, sin agrupar y cargando directos desde node_modules.
  • La Solución: Auditar sobre el build de producción (npm run build && npm run preview o Nginx en Docker).

2. Justificación de Soluciones Implementadas

FASE 1: Optimización de Assets

1. Conversión a WebP: Imágenes PNG gigantes (22 MB total) destruyeron el LCP. Usando sharp, se convirtieron a WebP/AVIF, reduciendo el tamaño a 830 KB (97% de ahorro).

2. Pre-carga del LCP: Se añadió <link rel="preload"> en el HTML y se reemplazó background-image por <img fetchpriority="high">.

3. Google Fonts: Se eliminó el @import bloqueante en CSS. Se movió a HTML con preconnect y preload, reduciendo de 14 a 8 variantes.

FASE 2: Compilación y Bundling (Vite)

1. Manual Chunks: Modificación de vite.config.ts para agrupar librerías estables en vendors separados (react, query, ui, chart).

2. Code-Splitting: Recharts (1.17 MB) se aisló y cargó dinámicamente con React.lazy(). Las animaciones de Framer Motion en el Hero se reemplazaron por CSS @keyframes.

FASE 3: Estabilidad Visual (CLS)

1. Dimensionamiento: Se agregaron width, height, loading="lazy" y decoding="async" a imágenes de catálogos.

2. Skeletons: Se implementó un componente PageLoader para evitar el renderizado de null en rutas administrativas.

FASE 4: Servidor de Producción (Nginx)

1. Compresión Gzip: Configuración de gzip_static on y pre-compresión con vite-plugin-compression2. Se corrigió bug en el Dockerfile.

2. Caché Eficiente: Cabeceras Cache-Control: public, max-age=31536000, immutable para la carpeta /assets.

FASE 5: Accesibilidad y SEO

1. Contraste WCAG AA: Ajuste de colores en badges (#c44a1a), textos secundarios (text-stone-500) y tooltips.

2. SEO y robots.txt: Se simplificó a Allow: / para no perjudicar auditorías en rutas protegidas.

4. Guía para Re-auditar el Rendimiento

  1. 1
    Construir el contenedor Docker

    Compila y expone la app optimizada en http://localhost:5173.

    docker compose up -d --build frontend
# O construye toda la aplicación:
docker compose up -d --build
  2. 2
    Abrir Chrome en Modo Incógnito

    Presiona Ctrl + Shift + N para iniciar una pestaña limpia de extensiones.

  3. 3
    Iniciar Sesión Previamente

    Navega a la URL e inicia sesión con una cuenta de prueba para acceder a rutas privadas.

  4. 4
    Ejecutar Lighthouse

    Presiona F12, ve a la pestaña Lighthouse, elige el dispositivo y haz clic en Analyze page load.
